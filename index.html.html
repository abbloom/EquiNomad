🚀 Publier sur GitHub Pages — guide complet

5 étapes simples pour mettre l'app en ligne gratuitement. Aucune compétence technique requise.

1
Créer un compte GitHub
Va sur github.com → Sign up → choisis un nom d'utilisateur (ex : nomadabloom) → valide ton adresse email.
2
Créer un dépôt (repository)
Une fois connectée → clic + en haut à droite → New repository
Nom : equi-training · Visibilité : Public · Coche ✅ Add a README file → Create repository
3
Uploader le fichier index.html
Dans ton dépôt → Add file → Upload files
Glisse-dépose le fichier index.html téléchargé
→ Commit changes ✅
💡 Le fichier doit s'appeler exactement index.html en minuscules
4
Activer GitHub Pages
Dans ton dépôt → onglet Settings → menu gauche Pages
Sous Branch → sélectionne main → dossier / (root) → Save
Attends 1-2 minutes ☕
5
🎉 Ton app est en ligne !
Ton URL : https://TON_USERNAME.github.io/equi-training/
Remplace TON_USERNAME par ton nom GitHub.
Partage ce lien — fonctionne sur mobile sans installation !
📱 Sur iPhone : Safari → icône partage → Sur l'écran d'accueil → l'app s'installe comme une vraie app
🔄 Mettre à jour les questions
Pour ajouter des questions après publication :
① Dans GitHub → clic sur index.html
② Clic ✏️ (icône crayon) en haut à droite
③ Trouve const QB = [ et ajoute tes questions à la fin du tableau
④ Commit changes → live en 30 secondes ✅
💡 Modèle de question QCM :
{g:6,topic:'g6-urg',icon:'🚑',tl:'Urgences',type:'qcm',
q:"Ta question ?",a:["A","B","C","D"],c:0,e:"Explication…"},
❓ Problèmes fréquents
L'URL affiche une page 404
Attends 2-3 min après activation de Pages. Vérifie que le fichier s'appelle exactement index.html à la racine du dépôt.
Les modifications n'apparaissent pas
Vide le cache (Ctrl+Shift+R sur PC) ou utilise la navigation privée. GitHub Pages met parfois 1-2 min à propager les changements.
Je veux un vrai domaine (ex: equitraining.fr)
Achète un domaine sur OVH ou Gandi (~10€/an), puis dans Settings → Pages → Custom domain. Compatible avec GitHub Pages.
